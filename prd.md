# Earn Compass PRD

## 1. 文档目的

本文档基于当前代码实现，重新定义 Earn Compass 的产品范围、核心流程与业务口径。它替代旧版 `PRD.pdf` 中已经过时的描述，作为后续产品、设计与开发协作的统一基线。

## 2. 产品概述

Earn Compass 是一个面向 CeFi / DeFi 收益型资产管理场景的 Web 应用，帮助用户记录投资仓位、跟踪收益表现、查看历史快照，并通过仪表盘和分析页理解组合变化。

当前产品已经具备以下能力：

- 投资记录管理：新增、编辑、删除、提前结束
- 收益看板：展示活跃仓位、收益汇总、APR 等核心指标
- 分析能力：图表化查看收益趋势、APR 分布、项目占比、波动情况、历史快照
- 账户系统：邮箱密码注册登录，Google / GitHub OAuth 登录
- 访客体验：未登录用户可直接浏览只读预览数据
- 设置中心：个人资料、语言、显示币种、数据导出、清空数据

## 3. 目标用户

- 需要集中管理多个平台收益仓位的个人投资者
- 同时参与交易所理财、链上借贷、LP、CeDeFi 产品的用户
- 希望以轻量方式记录本金、APR、收益和到期状态的用户

## 4. 产品目标

### 4.1 核心目标

- 降低多平台收益仓位的记录和维护成本
- 用统一口径展示当前收益表现与组合结构
- 为用户沉淀可回溯的投资快照，支持长期复盘

### 4.2 非目标

- 不提供真实资金托管或链上交易执行
- 不做自动下单、资产归集、钱包签名等能力
- 当前版本不包含管理员后台、团队协作、多租户体系

## 5. 核心使用场景

### 5.1 游客浏览

未登录用户进入首页后，可以直接查看一组内置的示例投资数据和分析图表，用于快速理解产品价值，但不能创建、编辑或删除记录。

### 5.2 注册并开始记录

新用户可以通过邮箱密码注册，也可以通过 Google / GitHub OAuth 登录。注册成功后即自动登录，并进入真实数据环境。

### 5.3 管理投资仓位

登录用户可录入以下信息：

- 项目名称
- 资产名称
- 项目链接
- 投资类型
- 投入金额与币种
- 开始时间、结束时间
- 预期 APR、实际 APR、预估/最终收益
- 描述与备注

### 5.4 结束或删除仓位

- 对已建立仓位，用户可以编辑或提前结束
- 提前结束时可补录结束日期、实际 APR、最终收益、备注
- 删除采用强确认机制，必须输入 `DELETE`

### 5.5 查看收益分析

用户可在分析页查看：

- 收益概览
- 快照趋势
- 投资收益趋势
- 组合收益波动
- APR 对比/分布
- 项目配置占比

### 5.6 数据导出与清理

用户可在设置页导出当前投资数据为 JSON，也可以清空当前数据源中的投资记录。

## 6. 信息架构

### 6.1 Dashboard `/`

首页聚焦实时组合总览与投资列表：

- 顶部导航
- 看板统计卡片
- 投资筛选器
- 活跃 / 历史投资列表
- 新增 / 编辑投资表单

### 6.2 Analytics `/analytics`

分析页聚焦趋势与快照：

- 收益概览
- 真实快照趋势
- 收益曲线
- 组合波动
- APR 分布
- 项目拆分

### 6.3 Settings `/settings`

设置页聚焦偏好与账户：

- 个人资料
- 通知开关占位能力
- 语言与显示币种
- 数据导出
- 清空数据

### 6.4 Auth `/login` `/register`

- 邮箱密码登录 / 注册
- Google OAuth
- GitHub OAuth

## 7. 角色与权限

### 7.1 游客

- 可访问 Dashboard、Analytics、Settings 页面
- 看到的是预览数据或本地默认内容
- 无法执行新增、编辑、结束、删除真实投资记录

### 7.2 已登录用户

- 只能访问和操作自己的远程投资数据
- 可创建、编辑、删除、结束自己的记录
- 可查看和手动生成自己的分析快照

## 8. 关键业务对象

### 8.1 Investment

投资记录是系统核心对象，包含：

- 基础信息：项目、资产、URL、类型、币种、备注
- 时间信息：开始时间、结束时间、持有天数
- 金额信息：本金、总收益、日/周/月/年收益
- 指标信息：预期 APR、实际 APR
- 状态信息：`ONGOING`、`ENDED`、`EARLY_ENDED`

### 8.2 Portfolio Snapshot

组合快照是某个日期下的聚合结果，用于分析趋势，包含：

- 总本金
- 组合预期 / 实际 APR
- 日 / 周 / 月 / 年收益
- 累计收益
- 活跃仓位数

## 9. 业务规则

### 9.1 投资状态

- `ONGOING`：进行中
- `ENDED`：正常结束
- `EARLY_ENDED`：提前结束

### 9.2 收益与 APR 口径

- 活跃仓位默认按预期 APR 推导日/周/月/年收益
- 已结束仓位优先使用手动录入的实际 APR 或最终总收益
- 当最终总收益存在时，系统会基于本金与持有天数反推实际 APR
- 持有天数最少按 1 天计算，避免 0 天异常

### 9.3 删除规则

- 删除不是游客能力
- 删除前需输入 `DELETE` 做二次确认
- 单条删除为软删除
- 设置页“清空全部数据”会清除当前用户的数据集合

### 9.4 预览模式

- 未登录时自动进入预览模式
- 预览模式使用内置 mock 数据
- 预览模式只读，所有写操作入口都应禁用或拦截

## 10. 非功能需求

- 支持中英文界面切换
- 支持多显示币种切换
- 页面需要兼容桌面端和移动端
- 核心页面交互反馈及时，表单与操作需要给出成功/失败提示
- 生产环境部署在 Cloudflare Workers，并需支持定时快照写入与到期提醒邮件
- 生产数据源为 PostgreSQL；部署平台迁移不要求同步迁移数据库

## 11. 当前版本边界说明

与旧文档相比，当前代码体现出的实际边界如下：

- 已实现的是“预览模式 + 登录后真实数据”，不是完整成熟的前台存储模式切换产品
- 已接入 PostgreSQL，而不是旧文档中的 SQLite
- 已实现自定义会话鉴权与 OAuth，而不是 NextAuth
- 已经具备分析快照和定时任务，不再属于纯规划能力
- 当前部署基线为 Cloudflare Workers + OpenNext，而不是 Vercel
