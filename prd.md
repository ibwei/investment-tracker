# Earn Compass PRD

## 1. 文档目的

本文档基于当前代码实现，定义 Earn Compass 的产品范围、核心用户流程、业务对象和版本边界。它用于替代旧版 PRD 中关于 SQLite、NextAuth、Vercel 优先部署、TradingView 或完整本地/远程双模式产品的过时描述。

## 2. 产品概述

Earn Compass 是一个面向 CeFi / DeFi 收益型资产和个人资产净值管理场景的 Web 应用。用户可以集中记录收益型投资仓位，跟踪本金、APR、收益、到期时间和组合变化；也可以聚合 CEX API、链上地址和手动资产，查看总资产、资产分布、同步状态和历史趋势。

当前产品已具备：

- 投资记录管理：新增、编辑、删除、提前结束
- 收益看板：展示活跃仓位、收益汇总、APR 等核心指标
- 分析能力：查看收入概览、真实快照趋势、收益趋势、组合波动、APR 分布和项目占比
- 资产聚合：CEX 来源、EVM 链上来源、手动资产、总资产摘要、分布、趋势、健康状态和明细
- 账户系统：邮箱验证码注册、邮箱密码登录、Google / GitHub OAuth 登录
- 访客体验：未登录用户可浏览只读预览数据
- 设置中心：个人资料、时区、语言、显示币种、通知开关占位、数据导出、清空数据
- 定时任务：投资快照、投资自动结算、到期提醒邮件、资产来源同步

## 3. 目标用户

- 需要集中管理多个平台收益仓位的个人投资者
- 同时参与交易所理财、链上借贷、LP、CeDeFi 产品的用户
- 希望用轻量方式记录本金、APR、收益和到期状态的用户
- 希望聚合 CEX、链上钱包和手动资产净值的用户
- 需要定期回顾收益变化、资产结构和到期安排的用户

## 4. 产品目标

### 4.1 核心目标

- 降低多平台收益仓位的记录和维护成本
- 用统一口径展示当前收益表现、组合结构和资产净值
- 沉淀可回溯的每日投资快照和资产快照，支持长期复盘
- 在投资到期前后提醒用户及时处理仓位
- 为未来“总资产 vs 投资本金 vs 收益”的融合分析打基础

### 4.2 非目标

- 不托管真实资金
- 不执行链上交易、钱包签名、自动下单、提现、划转或资产归集
- 不要求用户提供钱包私钥或助记词
- 不提供投资建议或收益承诺
- 当前版本不包含管理员后台、团队协作或多租户组织体系
- 当前版本不做高频实时行情、高频资产刷新、税务报表或复杂成本价追踪

## 5. 核心使用场景

### 5.1 游客浏览

未登录用户访问产品页面后，系统自动进入预览模式。游客可以查看示例投资、收益看板、分析图表和资产总览，快速理解产品体验。

游客限制：

- 不能新增、编辑、结束或删除真实投资
- 不能添加 CEX API Key、链上地址或手动资产
- 不能触发真实资产同步
- 不能捕获真实快照或清空远程数据
- 设置页的真实写操作需要登录后才有意义

### 5.2 注册和登录

用户可以通过以下方式进入真实数据模式：

- 邮箱验证码 + 密码注册
- 邮箱密码登录
- Google OAuth
- GitHub OAuth

邮箱注册流程要求先请求 6 位验证码。验证码 10 分钟有效，同一邮箱 60 秒内不能重复发送。注册或 OAuth 首次登录后，系统创建用户，并使用远程数据模式保存真实数据。

### 5.3 管理投资仓位

登录用户可录入和维护以下投资信息：

- 项目名称
- 资产名称
- 项目链接
- 投资类型
- 投入金额
- 投资币种
- 投资描述
- 开始时间
- 结束时间
- 预期 APR
- 实际 APR
- 预估或最终收益
- 备注

支持的投资类型：

- `Interest`
- `LP`
- `Lending`
- `CeDeFi`

### 5.4 结束或删除仓位

用户可以将活跃仓位提前结束。提前结束时可补录：

- 结束日期
- 实际 APR
- 最终收益
- 备注

删除规则：

- 游客不能删除真实数据
- 单条删除需要强确认，用户必须输入 `DELETE`
- 单条删除当前为软删除
- 设置页“清空全部数据”当前会硬删除当前登录用户的全部投资记录

### 5.5 查看收益分析

用户可在 Analytics 页面查看：

- 收入概览
- 真实快照趋势
- 收益趋势
- 组合收益波动
- APR 分布
- 项目配置占比

未登录用户看到预览分析数据。登录用户的数据来自远程 API 和自己的投资记录；真实快照趋势来自 `portfolio_daily_snapshots`。

### 5.6 管理资产总览

Assets 页面用于聚合用户资产净值。登录用户可：

- 添加 CEX 资产来源
- 添加 EVM 链上地址来源
- 添加、编辑、删除手动资产
- 查看总资产、变化、来源数量、失败来源数量和最近同步时间
- 查看资产按来源类型和类别的分布
- 查看 Top 资产、余额明细、DeFi / 结构化仓位、资产趋势和同步健康状态
- 手动同步单个来源或全部来源

当前 CEX 来源支持：

- Binance
- OKX
- Bybit
- Bitget
- Gate
- HTX
- KuCoin

当前链上来源支持：

- OKX Web3 provider
- `AUTO`、`OKX_EVM`、`OKX_SOLANA`、`OKX_SUI`、`OKX_TRON`、`OKX_BITCOIN`、`OKX_TON` provider alias
- EVM、Solana、Sui、Tron、Bitcoin 和 TON 地址的 wallet token 与 DeFi positions

手动资产支持的类型：

- `CASH`
- `STOCK`
- `FUND`
- `TOKEN`
- `REAL_ESTATE`
- `OTHER`

资产总览采用美元净值口径：

```text
totalValueUsd =
  sum(asset_balances.value_usd) +
  sum(asset_positions.net_value_usd) +
  sum(manual_assets.value_usd where is_deleted = false)
```

### 5.7 资产同步和健康状态

资产来源同步规则：

- CEX 凭据加密保存，只用于只读余额查询
- On-chain 来源保存公开地址，不保存私钥
- 单个来源同步成功后覆盖该来源当前余额和仓位
- 单个来源同步失败时标记为 `FAILED`，记录错误摘要和 sync log
- 同步完成后捕获当前用户资产快照
- 手动资产不调用外部 adapter，新增、编辑或删除后直接捕获快照

Assets 页面首屏只加载轻量 summary；趋势、来源、手动资产、余额、健康状态等详情按 tab 懒加载。

### 5.8 设置偏好和数据管理

Settings 页面当前支持：

- 修改姓名
- 查看邮箱
- 修改时区
- 切换语言
- 切换显示币种
- 本地通知开关占位
- 导出当前投资列表为 JSON
- 清空当前登录用户投资数据

注意：邮箱当前不能修改；通知开关当前是前端状态，占位展示为主；到期提醒邮件的用户级偏好还没有持久化接入。

### 5.9 自动结算、快照和提醒

系统通过 Cloudflare Cron 执行后台任务：

- 每 12 小时自动检查已过结束时间的 `ONGOING` 投资，并更新为 `ENDED`
- 每 12 小时为远程用户捕获收益/组合快照
- 每 4 小时同步资产来源并捕获资产快照
- 每天 10:00 和 22:00（UTC+8）给有活跃投资的活跃用户发送投资到期提醒邮件

到期提醒邮件会优先列出未来 24 小时内到期的投资，并附带其他活跃投资摘要。

## 8. 关键业务对象

### 8.1 User

用户对象包含：

- 邮箱
- 密码 hash
- 姓名
- 角色
- 状态
- 存储模式
- 时区

当前注册和 OAuth 创建的新用户：

- `role = USER`
- `status = ACTIVE`
- `storage_mode = REMOTE`
- 默认时区为 `Asia/Shanghai`

### 8.2 Auth Account

OAuth 账号映射包含：

- 用户 ID
- provider
- provider account ID
- OAuth 邮箱
- avatar URL

当前支持：

- Google
- GitHub

### 8.3 Email Verification Code

邮箱验证码用于注册前验证邮箱所有权，包含：

- email
- code hash
- purpose
- expires at
- consumed at
- created at

当前 purpose 为 `REGISTER`。

### 8.4 Investment

投资记录是收益管理核心对象，包含：

- 基础信息：项目、资产、URL、类型、币种、描述、备注
- 时间信息：开始时间、结束时间、持有天数
- 金额信息：本金、总收益、日/周/月/年收益
- 指标信息：预期 APR、实际 APR
- 状态信息：`ONGOING`、`ENDED`、`EARLY_ENDED`
- 删除信息：是否删除、删除时间

### 8.5 Portfolio Snapshot

组合快照是某个日期下的用户收益组合聚合结果，包含：

- 总本金
- 组合预期 APR
- 组合实际 APR
- 日/周/月/年收益
- 累计收益
- 活跃仓位数

### 8.6 Asset Source

资产来源包含：

- 来源类型：`CEX` 或 `ONCHAIN`
- Provider
- 来源名称
- 公开引用，例如钱包地址
- 加密配置，例如 CEX API 凭据
- 状态：`PENDING`、`ACTIVE`、`FAILED`、`DISABLED`
- 最近同步时间和错误摘要

### 8.7 Asset Balance

资产余额表示 CEX 或钱包 token 当前态，包含：

- 来源 ID
- 资产 symbol 和名称
- 数量
- USD 估值
- 类别：`SPOT`、`EARN`、`DEFI`、`CASH`、`OTHER`
- 原始数据摘要

### 8.8 Asset Position

资产仓位表示 DeFi 或结构化仓位当前态，包含：

- provider
- chain
- protocol id / name
- position type
- asset value
- debt value
- reward value
- net value
- 原始数据摘要

### 8.9 Manual Asset

手动资产表示用户自行维护的资产净值，包含：

- 名称
- 类型
- 数量或原始金额
- USD 估值
- 备注
- 软删除状态

### 8.10 Asset Snapshot

资产快照是某个日期下的用户总资产聚合结果，包含：

- 总资产 USD 净值
- 来源类型拆分
- 类别拆分
- 手动资产数量和净值

### 8.11 Scheduled Job Log

定时任务日志用于记录：

- 任务名称
- 执行日期
- 成功或失败状态
- 处理数量
- 耗时
- 错误信息
- 开始和结束时间

## 9. 业务规则

### 9.1 投资状态

- `ONGOING`：进行中
- `ENDED`：正常结束或自动结算结束
- `EARLY_ENDED`：提前结束

### 9.2 收益和 APR 口径

- 活跃仓位默认按预期 APR 推导收益
- 已结束仓位优先使用手动录入的实际 APR 或最终总收益
- 当最终总收益存在时，系统可基于本金和持有天数反推实际 APR
- 持有天数最少按 1 天计算，避免 0 天异常
- Dashboard 的加权 APR 基于活跃本金加权

### 9.3 投资删除规则

- 单条删除需要输入 `DELETE`
- 单条删除为软删除，记录设置 `is_deleted = true`
- 普通列表、快照和统计默认排除软删除记录
- 设置页清空全部投资当前为硬删除当前用户投资记录

### 9.4 资产来源规则

- 单用户资产来源上限为 10 个
- CEX 凭据必须加密保存
- On-chain 来源只保存公开地址
- 删除资产来源当前为硬删除，并通过外键级联删除该来源余额、仓位和同步日志
- `DISABLED` 来源不会被全量同步处理

### 9.5 手动资产规则

- 单用户手动资产上限为 50 个
- `amount` 和 `valueUsd` 必须为非负数
- 手动资产删除为软删除
- 手动资产不参与外部同步
- 手动资产写入或删除后立即捕获资产快照

### 9.6 预览模式

- 未登录时自动进入预览模式
- 预览模式使用内置 mock 数据
- 预览模式只读
- 写操作入口应禁用或被前端拦截

### 9.7 快照规则

- 投资手动快照只针对当前登录用户
- 投资 Cron 快照只处理 `storage_mode = REMOTE` 的用户
- 资产快照处理当前用户的自动余额、结构化仓位和未删除手动资产
- 同一用户同一天的组合快照使用 upsert
- 同一投资同一天的投资快照使用 upsert
- 同一用户同一天的资产快照使用 upsert
- 快照日期按应用时区生成

### 9.8 到期提醒规则

- 到期提醒只处理 `status = ACTIVE` 的用户
- 只统计 `ONGOING` 且未删除的投资
- 邮件按用户分组发送
- 未来 24 小时内到期的投资优先展示
- 当前实现是每天两次摘要，只要用户有活跃投资就发送，而不是仅在有 24 小时内到期项目时发送

## 10. 非功能需求

- 页面兼容桌面端和移动端
- 所有真实数据必须按用户隔离
- 所有写 API 必须要求登录
- Cron API 必须要求 `CRON_SECRET`
- 生产环境必须使用强 `AUTH_SECRET`
- CEX API 凭据必须使用强 `ASSET_CREDENTIAL_ENCRYPTION_KEY` 加密
- 数据库命令必须明确目标数据库
- 邮件发送失败应被记录到定时任务日志或验证码状态中
- UI 操作需要给出成功或失败反馈
- 生产部署目标为 Cloudflare Workers
- 正式数据源为 PostgreSQL
- 资产首屏必须保持轻量，不默认拉取所有明细、仓位和日志

## 11. 当前版本边界

与旧文档相比，当前代码体现出的真实边界如下：

- 已实现“游客预览 + 登录后远程数据”，不是完整成熟的 Local/Remote 双模式切换产品
- 已接入 PostgreSQL，不是 SQLite
- 已实现自定义 session、邮箱验证码注册和 OAuth，不是 NextAuth
- 已具备投资快照、自动结算、到期提醒邮件和 Cloudflare Cron
- 已具备 Assets 页面、资产来源、手动资产、资产快照、资产同步 Cron 和同步健康状态
- 当前部署基线为 Cloudflare Workers + OpenNext，不是 Vercel
- CEX 支持基础余额同步；链上当前主实现是 OKX Web3 wallet / DeFi API，覆盖 EVM 与部分异构链
- 通知开关还没有持久化到后端，也没有控制到期提醒发送策略
- 数据导入、找回密码、管理员后台、多组合视角仍未实现
