# Earn Compass PRD

## 1. 文档目的

本文档基于当前代码实现，重新定义 Earn Compass 的产品范围、核心用户流程、业务对象和版本边界。它用于替代旧版 PRD 中关于 SQLite、NextAuth、Vercel 优先部署或完整本地/远程双模式产品的过时描述。

## 2. 产品概述

Earn Compass 是一个面向 CeFi / DeFi 收益型资产管理场景的 Web 应用，帮助用户集中记录收益型投资仓位，跟踪本金、APR、收益、到期时间和组合变化。

当前产品已具备：

- 投资记录管理：新增、编辑、删除、提前结束
- 收益看板：展示活跃仓位、收益汇总、APR 等核心指标
- 分析能力：查看收入概览、真实快照趋势、收益趋势、组合波动、APR 分布和项目占比
- 账户系统：邮箱密码注册登录，Google / GitHub OAuth 登录
- 访客体验：未登录用户可浏览只读预览数据
- 设置中心：个人资料、时区、语言、显示币种、通知开关占位、数据导出、清空数据
- 定时任务：每 12 小时收益/组合快照、到期自动结算、每天两次到期提醒邮件

## 3. 目标用户

- 需要集中管理多个平台收益仓位的个人投资者
- 同时参与交易所理财、链上借贷、LP、CeDeFi 产品的用户
- 希望用轻量方式记录本金、APR、收益和到期状态的用户
- 需要定期回顾组合收益变化和到期安排的用户

## 4. 产品目标

### 4.1 核心目标

- 降低多平台收益仓位的记录和维护成本
- 用统一口径展示当前收益表现和组合结构
- 沉淀可回溯的每日组合快照，支持长期复盘
- 在投资到期前后提醒用户及时处理仓位

### 4.2 非目标

- 不托管真实资金
- 不执行链上交易、钱包签名、自动下单或资产归集
- 不提供投资建议或收益承诺
- 当前版本不包含管理员后台、团队协作或多租户组织体系
- 当前版本不做完整的链上资产自动同步

## 5. 核心使用场景

### 5.1 游客浏览

未登录用户访问首页后，系统自动进入预览模式。游客可以查看示例投资列表、收益看板和分析图表，快速理解产品体验。

游客限制：

- 不能新增投资
- 不能编辑投资
- 不能结束投资
- 不能删除真实投资记录
- 设置页的真实写操作需要登录后才有意义

### 5.2 注册和登录

用户可以通过以下方式进入真实数据模式：

- 邮箱密码注册
- 邮箱密码登录
- Google OAuth
- GitHub OAuth

注册或 OAuth 首次登录后，系统创建本地用户，并使用远程数据模式保存真实投资记录。

### 5.3 管理投资仓位

登录用户可录入和维护以下信息：

- 项目名称
- 资产名称
- 项目链接
- 投资类型
- 投入金额
- 投资币种
- 投资描述
- 开始时间
- 结束时间
- 预期 APR
- 实际 APR
- 预估或最终收益
- 备注

支持的投资类型：

- `Interest`
- `LP`
- `Lending`
- `CeDeFi`

### 5.4 结束或删除仓位

用户可以将活跃仓位提前结束。提前结束时可补录：

- 结束日期
- 实际 APR
- 最终收益
- 备注

删除规则：

- 游客不能删除真实数据
- 单条删除需要强确认，用户必须输入 `DELETE`
- 单条删除当前为软删除
- 设置页“清空全部数据”当前会清除当前登录用户的全部投资记录

### 5.5 查看收益分析

用户可在 Analytics 页面查看：

- 收入概览
- 真实快照趋势
- 收益趋势
- 组合收益波动
- APR 分布
- 项目配置占比

未登录用户看到预览分析数据。登录用户的数据来自远程 API 和自己的投资记录；真实快照趋势来自 `portfolio_daily_snapshots`。

### 5.6 设置偏好和数据管理

Settings 页面当前支持：

- 修改姓名
- 修改邮箱
- 修改时区
- 切换语言
- 切换显示币种
- 本地通知开关占位
- 导出当前投资列表为 JSON
- 清空当前登录用户投资数据

注意：通知开关当前是前端状态，占位展示为主；到期提醒邮件的用户级偏好还没有持久化接入。

### 5.7 自动结算和提醒

系统通过 Cloudflare Cron 执行后台任务：

- 每 12 小时自动检查已过结束时间的 `ONGOING` 投资，并更新为 `ENDED`
- 每 12 小时为远程用户捕获收益/组合快照
- 每天 10:00 和 22:00（UTC+8）给有活跃投资的活跃用户发送投资到期提醒邮件

到期提醒邮件会优先列出未来 24 小时内到期的投资，并附带其他活跃投资摘要。

## 7. 角色与权限

### 7.1 游客

游客可访问：

- Dashboard
- Analytics
- Settings
- Login
- Register

游客能力：

- 浏览内置示例投资
- 浏览预览分析图表
- 切换本地显示偏好

游客限制：

- 不能写入真实投资数据
- 不能捕获真实快照
- 不能清空远程数据

### 7.2 已登录用户

已登录用户能力：

- 创建、编辑、结束、删除自己的投资记录
- 查看自己的 Dashboard 和 Analytics
- 手动捕获自己的组合快照
- 更新自己的姓名、邮箱和时区

- 清空自己的投资数据

权限规则：

- 服务端 API 必须从 session 中读取 `userId`
- 用户只能访问和操作自己的数据
- OAuth 账号绑定不能跨用户泄露数据

## 8. 关键业务对象

### 8.1 User

用户对象包含：

- 邮箱
- 密码 hash
- 姓名
- 角色
- 状态
- 存储模式
- 时区

当前注册和 OAuth 创建的新用户：

- `role = USER`
- `status = ACTIVE`
- `storage_mode = REMOTE`
- 默认时区为 `Asia/Shanghai`

### 8.2 Auth Account

OAuth 账号映射包含：

- 用户 ID
- provider
- provider account ID
- OAuth 邮箱
- avatar URL

当前支持：

- Google
- GitHub

### 8.3 Investment

投资记录是系统核心对象，包含：

- 基础信息：项目、资产、URL、类型、币种、描述、备注
- 时间信息：开始时间、结束时间、持有天数
- 金额信息：本金、总收益、日/周/月/年收益
- 指标信息：预期 APR、实际 APR
- 状态信息：`ONGOING`、`ENDED`、`EARLY_ENDED`
- 删除信息：是否删除、删除时间

### 8.4 Portfolio Snapshot

组合快照是某个日期下的用户组合聚合结果，包含：

- 总本金
- 组合预期 APR
- 组合实际 APR
- 日/周/月/年收益
- 累计收益
- 活跃仓位数

### 8.5 Investment Daily Snapshot

单笔投资每日快照包含：

- 用户 ID
- 投资 ID
- 快照日期
- 本金
- 预期 APR
- 实际 APR
- 日/周/月/年收益
- 总收益
- 投资状态

### 8.6 Scheduled Job Log

定时任务日志用于记录：

- 任务名称
- 执行日期
- 成功或失败状态
- 处理数量
- 耗时
- 错误信息
- 开始和结束时间

## 9. 业务规则

### 9.1 投资状态

- `ONGOING`：进行中
- `ENDED`：正常结束或自动结算结束
- `EARLY_ENDED`：提前结束

### 9.2 收益和 APR 口径

- 活跃仓位默认按预期 APR 推导收益
- 已结束仓位优先使用手动录入的实际 APR 或最终总收益
- 当最终总收益存在时，系统可基于本金和持有天数反推实际 APR
- 持有天数最少按 1 天计算，避免 0 天异常
- Dashboard 的加权 APR 基于活跃本金加权

### 9.3 删除规则

- 单条删除需要输入 `DELETE`
- 单条删除为软删除，记录设置 `is_deleted = true`
- 普通列表、快照和统计默认排除软删除记录
- 设置页清空全部数据当前为硬删除当前用户投资记录

### 9.4 预览模式

- 未登录时自动进入预览模式
- 预览模式使用内置 mock 数据
- 预览模式只读
- 写操作入口应禁用或被 store 拦截

### 9.5 快照规则

- 手动快照只针对当前登录用户
- Cron 快照只处理 `storage_mode = REMOTE` 的用户
- 同一用户同一天的组合快照使用 upsert
- 同一投资同一天的投资快照使用 upsert
- 快照日期按用户时区生成

### 9.6 到期提醒规则

- 到期提醒只处理 `status = ACTIVE` 的用户
- 只统计 `ONGOING` 且未删除的投资
- 邮件按用户分组发送
- 未来 24 小时内到期的投资优先展示
- 当前实现是每天两次摘要，只要用户有活跃投资就发送，而不是仅在有 24 小时内到期项目时发送

## 10. 非功能需求

- 页面兼容桌面端和移动端
- 所有真实数据必须按用户隔离
- 所有写 API 必须要求登录
- Cron API 必须要求 `CRON_SECRET`
- 生产环境必须使用强 `AUTH_SECRET`
- 数据库命令必须明确目标数据库
- 邮件发送失败应被记录到定时任务日志
- UI 操作需要给出成功或失败反馈
- 生产部署目标为 Cloudflare Workers
- 正式数据源为 PostgreSQL

## 11. 当前版本边界

与旧文档相比，当前代码体现出的真实边界如下：

- 已实现“游客预览 + 登录后远程数据”，不是完整成熟的 Local/Remote 双模式切换产品
- 已接入 PostgreSQL，不是 SQLite
- 已实现自定义 session 和 OAuth，不是 NextAuth
- 已具备快照、自动结算、到期提醒邮件和 Cloudflare Cron
- 当前部署基线为 Cloudflare Workers + OpenNext，不是 Vercel
- 通知开关还没有持久化到后端，也没有控制到期提醒发送策略
- 数据导入、找回密码、管理员后台、多组合视角仍未实现
